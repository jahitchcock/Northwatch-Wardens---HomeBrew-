## Priority Labels Explained

- **Critical** (Phase 1-2 core): Campaign won't run without these
- **Very Important** (Phase 2): Major usability problems without these
- **Important** (Phase 2 DM notes): Adds significant value
- **Nice to Have** (Phase 3-4): Improves experience but not blocking

## Parallel Work Strategy

### Can Work Simultaneously:
- **All 6 Adventure DM Notes** (#2.6-2.11)
  - Different person per adventure
  - All follow same template
  - Independent of each other

- **Most Phase 3 Issues** (#3.1-3.6)
  - Once Phase 2 core (#2.1-2.5) is done
  - Mostly independent

- **Phase 4 Overviews** (#4.4-4.5)
  - Can work in parallel with #4.1-4.3

### Must Be Sequential:
1. **#1.1** (TOC reorg) should complete first
2. **#2.1-2.3** (Quick Start, At A Glance, First Session) before testing
3. **#4.6** (Final TOC) must be absolute last (integrates all new files)

## Testing Checkpoints

### ✅ After Phase 1 (#1.1):
- Build PDF and verify structure
- Check page count reduction

### ✅ After Phase 2 Core (#2.1-2.5):
- **Critical Test**: Can a new DM prep and run Session 0/1?
- Do they understand the campaign?
- Can they find what they need?

### ✅ After Phase 2 Complete (all #2.x):
- Full adventure run-through
- DM uses only the guide
- Collect usability feedback

### ✅ After Phase 4 Complete (all issues):
- Final comprehensive test
- Verify all cross-references
- Check final page count (<200 pages target)
